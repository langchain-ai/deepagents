# Simple rubric (0	3 each)

## 1) Correctness
- 0: does not run / wrong output
- 1: runs but fails common cases
- 2: mostly correct, minor edge cases
- 3: correct + handles edge cases

## 2) Readability
- 0: very hard to read
- 1: inconsistent naming/formatting
- 2: clear structure + naming
- 3: idiomatic, well-factored

## 3) Git hygiene
- 0: no commits or giant commit
- 1: some commits, unclear messages
- 2: small commits, decent messages
- 3: small commits + descriptive messages + branches used

## 4) Testing (when required)
- 0: none
- 1: minimal
- 2: covers typical cases
- 3: includes edge cases
